Overview of Html
What is html?
HTML (HyperText Markup Language) is used to structure and display content on the web. It allows developers to organize text, images, links, and multimedia elements into a coherent layout, enabling browsers to interpret and present this content to users. HTML provides the foundation for web pages by defining various elements such as headings, paragraphs, tables, forms, and more. It is essential for creating user-friendly websites, enabling interaction, and serving as the backbone for web development when combined with CSS and JavaScript.

Why do we use html?

What is the root element of html?
The root element of an HTML document is the `html` tag, which encloses all the content and structure of the webpage. What are href vs src attributes?
The `href` and `src` attributes are both used in HTML, but they serve different purposes. The `href` (hyperlink reference) attribute is primarily used in `a` (anchor) tags to link to another webpage or resource, allowing users to navigate to that location when the link is clicked. In contrast, the `src` (source) attribute is used in elements like `img`, `script`, and `iframe` to specify the location of an external resource, such as an image, JavaScript file, or another HTML document that should be embedded or displayed on the current page. Essentially, `href` is for linking, while `src` is for embedding content.

What is git vs github?
Git is a distributed version control system used to track changes in source code during software development. It allows multiple developers to collaborate, manage different versions of code, and revert to previous states if needed. GitHub, on the other hand, is a cloud-based platform that hosts Git repositories, offering features like issue tracking, pull requests, and collaboration tools. While Git operates locally, GitHub facilitates remote collaboration by providing a centralized location to store and manage Git repositories online.

What is the workflow of github?
GitHub's workflow typically begins with creating or cloning a repository, where developers manage and store their project's code. Users work on new features or bug fixes in separate branches, and once changes are complete, they commit and push them to GitHub. These changes are then reviewed through pull requests, where collaborators can comment and suggest improvements. Once approved, the code is merged into the main branch, maintaining a structured and collaborative development process.

What command do we need to use to move the changes to staged changes?
To move changes to the staged area in Git, use the `git add` command followed by the file name or a dot (`.`) to add all changes in the current directory. Staging means you are preparing the files for the next commit, but they aren't committed yet. For example, `git add .` will stage all modified or new files, while `git add file stages specific files. Once staged, the files are ready to be committed with a message using `git commit`.

How can you stage all files at a time?
To stage all files at once in Git, you can use the command `git add .` in your terminal or command line. This command stages all the modified, new, or deleted files in the current directory and its subdirectories for the next commit. It's a quick way to prepare multiple changes simultaneously. However, ensure you review the changes before committing to avoid unintentionally adding unwanted files.

What is the git push command and use of it?
The `git push` command is used to upload local repository content to a remote repository, typically hosted on platforms like GitHub or GitLab. It transfers committed changes, allowing others to access and collaborate on the updated project. This command is crucial in version control, ensuring that local changes are synchronized with the central repository. By using `git push`, developers can share code, track revisions, and work seamlessly in distributed teams.

What is git commit command usage?
The `git commit` command is used to save changes to the local repository after staging them with `git add`. It creates a snapshot of the current state of the project, allowing you to document changes with a descriptive message using the `-m` option. For example, `git commit -m "Commit message"` records the changes along with the message. Each commit represents a version in the project's history, which can later be reverted or reviewed as needed.

What is the use of the git init command?
The `git init` command is used to initialize a new Git repository in a project directory. It creates a hidden `.git` folder, where Git stores all the necessary configuration files and the project’s version history. This command enables tracking of changes and version control for the project. It's the first step in starting a Git-managed project, either for a new project or for adding Git to an existing one.

What are formatting tags?
Formatting tags in HTML are used to apply styles and structure to text within a web page. They enhance the presentation of content by defining how text is displayed, such as bold (`strong`), italic (`em`), underline (`u`), or strikethrough (`del`). These tags help improve readability and emphasize specific parts of the text. What are input tag attributes?
The `input` tag in HTML is used to create interactive controls in web forms for user data entry. Key attributes include `type`, which specifies the input's format (e.g., text, password, email); `name`, which identifies the input when the form is submitted; `value`, which sets the default value displayed in the input field; and `placeholder`, which provides a hint to the user about what to enter. Other attributes, such as `required`, `maxlength`, and `disabled`, further customize the behavior and appearance of the input fields, enhancing user experience and form validation.

What are form attributes?
Form attributes are specific properties that define how HTML forms behave and interact with users. They include attributes like `action`, which specifies the URL where the form data should be sent upon submission, and `method`, which determines how the data is sent (e.g., GET or POST). Other important attributes include `enctype`, which specifies the encoding type for form data, and `target`, which defines where to display the response (e.g., in a new window or the same frame). Together, these attributes help control the functionality and appearance of web forms, ensuring a better user experience.

What is a marquee tag?
The `marquee` tag is an HTML element used to create scrolling text or images across a web page. It allows for horizontal or vertical movement, with various attributes to control the speed, direction, and behavior of the scrolling content. Although it was once popular for adding dynamic text effects, the `marquee` tag is now considered obsolete in HTML5 and is not recommended for use in modern web design due to accessibility and usability issues. Instead, CSS animations and JavaScript provide more flexible and accessible alternatives for creating similar effects.

What is the marquee tag attributes?
The `marquee` tag is a non-standard HTML element used to create scrolling text or images across a webpage. Although it is deprecated and not recommended for use in modern web development, it has several attributes that control its behavior: 1. **direction**: Specifies the direction in which the content scrolls. Possible values include "left," "right," "up," and "down." 2. **scrollamount**: Defines the speed of the scrolling text. A higher value results in faster scrolling. 3. **scrolldelay**: Sets the delay between each scroll movement in milliseconds, controlling how quickly the text moves. 4. **behavior**: Determines how the content scrolls. Options include "scroll" (default), "slide" (stops at the end), and "alternate" (bounces back and forth). While the `marquee` tag can be fun for adding dynamic text to a webpage, it's advisable to use CSS animations or JavaScript for similar effects in a more accessible and standardized way.

Can we use block elements in inline elements?
In HTML, inline elements cannot contain block elements. Inline elements, such as `span`, `a`, and `strong`, are designed to flow within the content of a block-level element, such as `div`, `p`, or `h1`, without causing a line break. Conversely, block elements create a new line and take up the full width available. If you try to place a block element inside an inline element, it may not render correctly and could lead to unexpected results. To maintain proper HTML structure, always ensure that block elements are nested within other block elements.

What are difference between abs vs real paths?
An absolute path (abs path) refers to the complete directory structure from the root of the filesystem to a specific file or directory, providing a unique location regardless of the current working directory. In contrast, a relative path (real path) is defined in relation to the current working directory, making it more flexible but dependent on the user's current location in the filesystem. While absolute paths are always accurate and unambiguous, relative paths are often shorter and easier to use for navigating within a project. Understanding the distinction is crucial for effectively managing file systems and ensuring correct file access in programming and scripting.

What are summary and details tags?
The `summary` and `details` tags in HTML are used to create a collapsible content feature, enhancing user experience by providing an interactive way to present information. The `details` tag encapsulates a section of content that can be expanded or collapsed, while the `summary` tag defines a summary or heading that is always visible, serving as a trigger for expanding the hidden content. When a user clicks on the summary, the details expand to reveal more information, making it useful for FAQs, descriptions, or additional content without cluttering the page. This feature is natively supported in modern web browsers, promoting cleaner and more organized web layouts.

what are fieldset and legend tags?
The `fieldset` and `legend` tags in HTML are used to group related elements within a form. The `fieldset` element creates a visual grouping by drawing a box around the enclosed form elements, making the form more structured and easier to understand. The `legend` tag is placed within the `fieldset` to provide a caption or title for the group, giving context about the grouped elements. Together, these tags enhance form accessibility and usability by clearly indicating the purpose of a set of form fields.
